# Dogfood — offline co-pilot loop (Layer B prototype)

**UX contract:** Predict → Ask → Explain (never “AI plays the gig”).  
**X alignment:** co-pilot *where you work* — here, greppable fixtures until QML chrome lands.

## Loop

```text
session ontology  (+ optional session-mirror.v1.json)
        ↓
  tools/exo/copilot_why_next.py   ← ranking + reasons (offline)
        ↓
  results/COPILOT-WHY-NEXT.md     ← Explain
  intent-inbox.v1.json            ← Predict (status: proposed)
        ↓
  human ack                       ← Ask
        ↓
  [future] reconciler → ControlObject (P-06) → engine
```

## Rules

- Offline fixtures only — **not** wired to the live Mixxx engine yet.
- Live writes must go through **one ControlObject writer** (`P-06`).
- RT thread must never parse JSON or run the co-pilot (`P-02`).
- Spotify rows: `multi_deck_allowed=false` → proposals stay **sequence-only**.
