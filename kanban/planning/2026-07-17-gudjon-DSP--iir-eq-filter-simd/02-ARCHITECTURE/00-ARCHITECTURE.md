# Architecture

## The measurement design (Wave 1)
A new benchmark translation unit in the `mixxx-test`/benchmark tree (sibling to
`src/test/waveformrenderbenchmark.cpp`) instantiates `EngineFilterIIR<5, IIR_BP>`
(`EngineFilterBiquad1Peaking`-shaped) and `EngineFilterIIR<8, IIR_LP>`
(`EngineFilterBessel8Low`/`EngineFilterButterworth8Low`-shaped) directly, calls `setCoefs(...)` once
outside the timed loop (coefficient design is not part of the per-buffer cost this PS measures), then
times repeated `process(pIn, pOutput, bufferSize)` calls over a 1024-frame (2048-sample, interleaved
stereo) synthetic buffer at 44.1kHz, generated by a seeded LCG (same determinism discipline as
`waveformrenderbenchmark.cpp`'s synthetic waveform). Percentiles/label/`DoNotOptimize` follow that
file's established pattern exactly — no new benchmark-reporting convention is introduced.

Because `process()` touches only `double`/`CSAMPLE` arrays in plain CPU memory — no GL context, no
Qt event loop, no `ControlObject` — this bench needs no offscreen surface and no skip path: every run
yields a complete p50/p90/p99/max distribution.

## The optimization design (Wave 2, high-level; the concrete diff is execution-time, not scoping)
The `process()` public entry point and the ramping branch (lines 245-291) are untouched. Only the
steady-state (`!m_doRamping`) loop body (lines 241-244) is replaced, per `(SIZE, PASS)` instantiation,
with either:
- an Accelerate `vDSP_biquad`/`vDSP_deq22` call over a one-time-repacked cascade (biquad-shaped
  instantiations), or
- a hand-rolled NEON path processing `m_buf1`/`m_buf2` (L/R) in parallel SIMD lanes (higher serial
  orders).

Any repacked coefficient/delay-line state lives in new member storage set up once in `setCoefs`/
`setCoefs2` (which already run off the RT thread on parameter change, per the class's existing
contract) — `process()` itself remains allocation-free and lock-free, per `P-02`/`P-16`. This
preserves `EngineFilterIIR`'s existing Qt affinity (may emit, never receive — `P-20`) and its
single-writer `ControlObject` contract upstream (`P-06`), neither of which this internals-only change
touches.

## Patterns / ADRs cited
`P-02` (RT no-alloc), `P-03` (benchmark-as-contract), `P-16` (lock-free RT handoff), `P-18` (p99 not
mean), `P-24` (arm64-native build), `P-25` (pin the baseline), `AP-02` (speedup that regresses house
physics), `AP-11` (mean hides the underrun). No ADR is needed — this dossier does not make a
path-pinning architectural decision; it optimizes an existing internal implementation behind an
unchanged public interface.
