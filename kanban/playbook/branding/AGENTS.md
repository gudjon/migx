# Branding — agent rules

## EARS triggers

- THE agent SHALL cite `BRND-*` IDs when using branding concepts from this playbook.
- WHEN the task involves messaging, verbal identity, visual brand direction, brand systems, rollout, or brand governance, THE agent SHALL load `kanban/playbook/branding/BRND-playbook.md` as primary process reference.
- WHILE market position, audience, category, or differentiated value is unresolved, THE agent SHALL complete **BRND-P1 / BRND-DIY-1** (and consult live claims in `migx-brand-positioning-experience-designer.md`) **before** recommending logos, palettes, or tagline spam.
- WHERE product UX, interface expression, or in-app chrome is the brand problem, THE agent SHALL cross-reference `res/design/DESIGN.md` and NextGen modes knowledge — not invent a parallel design system.
- WHERE customer evidence or discovery is required, THE agent SHALL cross-reference `product-discovery-customer-leadership-migx.md` (DC-PDCL) and owner-DJ evidence notes.
- WHERE perception/adoption measurement is required, THE agent SHALL propose a closed loop (capture + threshold) rather than vibes-only “brand feel.”
- IF asked for a short operating checklist only, THE agent SHALL use this file + DIY ladder in the playbook — not dump external brand essays into the repo.
- IF the task would rename packages, product binary, or public mark, THE agent SHALL stop for **owner value judgment** (irreversible brand act).

## Cite, don't restate

- Live **product positioning claims** → `migx-brand-positioning-experience-designer.md`
- Live **company strategy** → `Strategy-Current.md`
- Process / how to brand → this directory (`BRND-*`)
