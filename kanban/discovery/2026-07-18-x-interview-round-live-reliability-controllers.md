# X “interview” round 2 — live reliability, firmware, gear path

## E16 — Empty USB at set start; freestyle from 6-month backup (past_behavior)
**Who:** @tynanofficial (2025-12, ~6k likes, 100k+ views)  
**Said:** After 10 years DJing, loaded USB to start set — nothing there. Backup USB only had one playlist from 6 months ago; freestyled entire set from that. Explicit reminder: always backup, bring extra USBs.  
**Reply (field knowledge):** @ReiHerrera — known CDJ-3000 firmware issue; carry **two USBs** (one for older CDJs, one for newer).  
**Implication:** Worst-case ritual is **media redundancy + stale-crate salvage**, not AI next-track. Product that verifies export integrity / dual-export for firmware forks matches lived pain.

## E17 — Single USB = unprofessional; CDJs don’t link (past_behavior / social norm)
**Who:** @Solphendukaa (2026-01, ~1k likes, 126k views)  
**Said:** A-list DJs still arrive and discover CDJs don’t link; walking with only one USB of updated music is like driving without a spare wheel. Replies praise Deep House prep culture and “bag of CDs” era professionalism.  
**Implication:** Social enforcement of **dual media** is already cultural. Opp-D “share” in the field = spare stick + prep discipline, not invite-to-NAS.

## E18 — 1/5 tracks corrupted mid-night; old backup USB (past_behavior)
**Who:** @dirtmonkeymusic (2026-02)  
**Said:** ~1 of every 5 tracks said corrupted that night; loaded old backup USB; suspects new CDJ firmware again.  
**Implication:** Corruption is **partial and ongoing**, not only total empty stick — need per-track integrity + emergency crate, not only full-library re-export.

## E19 — Frantic USB re-export 10 min before set (past_behavior)
**Who:** @dirtmonkeymusic (2025-11)  
**Said:** Frantically updating USBs so they work on new firmware 10 minutes before set. Companion post: CDJ firmware drama; stay safe if you never update rekordbox.  
**Thread replies:** @gspacebass never uses rekordbox — raw folders + BPM-in-filename sort as backup plan; @KendamaBrad hasn’t updated in 6 years; dual-version rekordbox workaround discussed.  
**Implication:** **Update surface is the hazard.** Migx must not force last-minute cloud/OS churn; offline export + stable format > feature churn.

## E20 — Blank USBs after CDJ update without latest rekordbox (past_behavior + public warning)
**Who:** @Modestep (2025-11, high engagement)  
**Said:** New CDJ update dumbest thing; if you haven’t updated rekordbox, USBs show **blank** on new CDJs and updated 3000s; wants a better standard.  
**Implication:** Industry lock-in creates blank-set nightmares. Open/portable crate pack + dual-export path is a real wedge (aligns EXO session export / dual-USB assist thesis).

## E21 — Festival sidestage USB test desks (past_behavior / venue investment)
**Who:** @weraveyou citing @dannic / Tomorrowland (2025-07)  
**Said:** Tomorrowland provided DJ setup for artists to **test USBs** before performing.  
**Implication:** Top venues treat **verify-before-stage** as infrastructure. Migx prep co-pilot / export checklist is productizing what festivals already staff.

## E22 — Two-USB non-negotiable after mid-set fail (past_behavior)
**Who:** @vioLit_TV (2026-04, working DJ/creator)  
**Said:** Non-negotiable for every gig: at least two USBs. Once one fails mid-set, never risk it again.  
**Also:** @shangatatu (2026-04) — venue USB refused to read; old backup USB saved the set. @pshift6c (2026-07) — digital, enough tracks + backup USB.  
**Implication:** Dual-USB is **learned from failure**, not theory. Feature discovery energy is spent on survival kits.

## E23 — Gear path: controller PCDJ → club USB/CDJ migration pain (past_behavior)
**Who:** Japanese working DJs trending “gear history” discourse (2026-07): @shiro09, @Remi_nyaaaan, @416hatoren, @SYNDROME_RRG  
**Said (aggregate):**  
- Started Virtual DJ / DDJ → now club standard is rekordbox + Pioneer CDJ USB; migration “めちゃくちゃ面倒” (extremely annoying).  
- Advise beginners buy FLX4/400 path aimed at USB/CDJ, not pure PCDJ dead-end.  
- Serato single-USB cable venues exist; Traktor DVS lock-in + rekordbox sub cost friction when trying USB backup.  
- Ribbon controllers valued for **cue hitting** — hardware cue UX matters.  
**Implication:** Opp-C beachhead pain is **export/interop and cue-authoring on controller path**, not multi-source stream mix. Controller map + cue export > dark-UI speculation this pass (thin pure “can’t see screen” DJ signal).

## E24 — Stage crash: laptop off stage, mixer dies; show continues (past_behavior)
**Who:** @NinoAndrettiii (2026-07)  
**Said:** Laptop flew off stage, mixer took a shit; still had a great show.  
**Implication:** Hardware violence is real; **resilience / degrade paths** matter more than AI features when physical fails. Supports reliability north-star (MTL/soak) as table stakes.

## E25 — AI setlist ad rejected by scene taste (past_behavior / strong preference signal)
**Who:** @saluteAUT (2025-07, ~1.2k likes)  
**Said:** Served ad for AI tool that builds DJ setlist from prompts for dance music; “cannot stress how much you need to not be djing if the bare minimum is too much for you to do by yourself.”  
**Implication:** **Public identity** of many dance DJs rejects automated set construction. Co-pilot must be **assist + explain + dismiss**, never “I made your set.” No past_behavior of live AI Ack found this pass either.

## E26 — “AI DJ” discourse is Spotify consumer, not pro decks (observation / negative evidence)
**Who:** Recent X stream (2026-07): Spotify AI DJ “lost the plot,” casual listeners using AI DJ for discovery — **not** working DJs Acking mid-CDJ set.  
**Also:** AI music-generation ads dominate search noise; filtered out as hype.  
**Implication:** Do not confuse consumer AI radio with Opp-A. Still **zero** first-person “I accepted AI next-track on CDJs mid-gig.”
