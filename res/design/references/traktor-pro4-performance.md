# Reference analysis — Traktor Pro 4, Performance view (4-deck)

Third industry reference for the NextGen design gate. Traktor is the multi-deck / internal-mixer / stems
leader, and the source of the single most actionable learning for us: **colour-coded keys**.

## What's on the screen
- **Top bar** — logo, CPU meter, master clock + level, layout selector (Mixer view), settings. Center
  master-tempo panel: SNAP · QUANT · **128.00** master BPM · LINK · AUTO · MASTER · BOOST.
- **4-deck grid (A/B/C/D, 2×2)** — Traktor shows four decks comfortably; it is *the* multi-deck app.
- **Deck headers (compact)** — title · **−remaining** · **BPM** · deck letter, on one tight line. SYNC /
  MASTER. Minimal chrome per deck → that's how four fit.
- **RGB horizontal waveforms** + overview strip with colour-coded hot-cue markers (1–4).
- **Stem deck** (deck D) — four stem lanes **DRUMS / BASS / OTHER / VOCALS**, each its own waveform +
  controls. Stems are first-class (2024–25 trend, shared with rekordbox 7).
- **Internal center mixer** (Traktor signature) — per channel GAIN · HI/MID/LO EQ · FLTR · FX1/2 · **KEY**
  · headphone cue · fader. The software *is* the mixer (vs Serato leaning on hardware).
- **Browser (Track Collection)** — #, cover, **TITLE · ARTIST · BPM · KEY · RATING · FILE NAME**.

## Adopt — colour-coded KEY is the headline
- **★ Colour-coded KEY.** The KEY column uses Open-Key notation (`7m`, `10m`, `6m`, `9d`, `12m`…) **with a
  colour per key**, so harmonically-compatible tracks are *scannable by colour* — the DJ finds a matching
  mix without reading. This is a large cognitive-load reducer and the concrete upgrade to our KEY badge:
  **colour the key by its wheel position** (Camelot/Open-Key), on the deck, ARRANGE rows, and co-pilot chips.
- **4-deck grid → validates our multi-deck (4–6) flexibility.** Design the deck module to **tile** (2/4/N)
  from the start; compact headers are what make it fit (matches "minimal chrome").
- **Compact deck header** — title · −remaining · BPM in one tight line. The low-load anchor done right.
- **Stems first-class** — a later stem module (DRUMS/BASS/OTHER/VOCALS lanes + mutes).
- **Internal mixer** — relevant for laptop/controllerless DJing (our EQ/mixer module), incl. a per-channel
  KEY (pitch-shift-to-key) control.

## Reject / diverge
- The full 4-channel knob mixer is **knob-dense**. We keep the mixer **mode-scoped / on-demand** and let the
  co-pilot carry some of the harmonic reasoning rather than surfacing every knob at all times.
- Power-user columns (FILE NAME, many toggles) stay **opt-in**, not default — low-load first.

## Our stance
Adopt **colour-coded key + multi-deck tiling + compact headers now** (they inform the deck-track-identity
module and the deck layout). Internal mixer and stems become their own later modules.
